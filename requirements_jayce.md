# CarView2 图像预览与异常抓拍需求

## 目标

保留 CarView2 的实时图像预览能力，同时接通异常抓拍流程。所有从开发板传来的图像都要在“图像预览”窗口出现；只有 GD32 判定为有目标的图像，才进入“异常抓拍画面”和“异常信息列表”。

## 当前判定规则

- GD32 发送普通图像包：`type=0x01, format=0x01`，内容为 RGB565 图像分片。
- GD32 发送检测结果包：`type=0x02, format=0x02`，内容为 DET_V2。
- Linux 开发板按 `frame_id` 合并图像和 DET_V2 检测结果。
- 有 DET_V2 目标：该图像视为异常图像。
- 无 DET_V2 目标：该图像视为正常图像。

## 显示规则

1. 正常图像：
   - 显示到 CarView2 的“图像预览”。
   - 不生成异常记录。
   - 不进入“异常抓拍画面”。
   - 不进入“异常信息列表”。

2. 异常图像：
   - 显示到 CarView2 的“图像预览”。
   - 生成异常记录。
   - 显示到“异常抓拍画面”。
   - 加入“异常信息列表”。
   - 在地图上显示异常点。

## 数据链路

```text
GD32
  ├─ IMAGE RGB565 包
  └─ DET_V2 检测包
        │
        ▼
i.MX6ULL / gd32_bridge
  ├─ 读取 /dev/ttyACM0
  ├─ 按 frame_id 重组图像
  ├─ 按 frame_id 缓存 DET_V2
  └─ TCP:8766 发送到 CarView2
        │
        ▼
CarView2
  ├─ 所有图像更新图像预览
  └─ 有检测目标的图像生成异常记录
```

## TCP 协议约定

- `TYPE_RGB565 = 0x03`：普通 RGB565 图像。
- `TYPE_RGB565_ANOMALY = 0x04`：异常 RGB565 图像。
- `TYPE_RGB565_ANOMALY` payload 格式：

```text
[TCP 52B header][CdcDetObjectV2 28B][RGB565 image][CRC16 2B]
```

其中 `IMAGE_LEN` 只表示 RGB565 图像字节数，不包含 `CdcDetObjectV2` 的 28 字节。

## 当前固定调试环境

- 上位机/Windows IP：`172.20.10.3`
- 开发板 IP：`172.20.10.12`
- 开发板部署目录：`/home/root/`
- CarView2 TCP 端口：`8766`

## 验证命令

测试发送器混合发送：

```bash
cd /home/root
./tcp_image_test_sender 172.20.10.3 8766 30 200 mixed
```

真实桥接程序：

```bash
cd /home/root
./gd32_bridge 172.20.10.3 8766
```

## 验收标准

- `normal` 测试模式下，CarView2 只更新图像预览，不新增异常列表记录。
- `anomaly` 测试模式下，CarView2 更新图像预览，并持续新增异常列表记录。
- `mixed` 测试模式下，CarView2 每帧都更新图像预览，只有异常帧进入异常抓拍和异常列表。
- 真实 GD32 桥接时，有检测目标的帧进入异常抓拍，无检测目标的帧只进入图像预览。
